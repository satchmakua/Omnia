# DECISIONS.md — Decision Log

Settled decisions and *why*. **Do not relitigate these.** If you think one is wrong, write the concern into `PROGRESS.md` for review — don't silently reverse it. Add new decisions as they're made, with date and status.

Format: **ID — title** · status · rationale.

---

**D1 — Working title is "Omnia — the everything simulator."** · accepted
The name reflects the ambition; scope is nonetheless bounded (see D2).

**D2 — Scale: a town of hundreds of deep agents.** · accepted
Not a city, nation, or world. Rationale: depth and believability per agent beat raw population. This keeps every living agent fully simulated and shifts the hard problem from spatial abstraction to *temporal* history compression (D4). Explicitly rules out aggregate off-screen population / "Schrödinger's citizens." Changing scale is a deliberate future decision, not something to build for preemptively.

**D3 — Language: TypeScript end-to-end (simulation core and UI).** · accepted
One language minimizes maintenance surface for an AI working across many sessions; the type checker guards against editing mistakes; the browser makes the UI easy and shareable. At town scale TS is fast enough for the core, so we deliberately avoid a second high-performance language (Rust/Python). Revisit only if scale (D2) changes.

**D4 — History model: smart lossy compression, "fidelity ∝ importance × recency."** · accepted
History is central to the concept but must stay cheap. Recent/important is stored sharply; old/trivial dissolves into statistics and one-line legends. Implemented via multi-resolution memory, importance scoring + pruning, a tiered Chronicle, statistical strata, tombstones, and optional procedural regeneration (see `SIMULATION_MODEL.md`). Chosen over both "keep everything raw" (unbounded growth) and naive aggressive forgetting (flattens the story). Guard against flattening — Legends must read like a story, not a spreadsheet.

**D5 — Engine architecture: ECS, deterministic seeded core, sim/render decoupled, layered agent brain.** · accepted
ECS lets each session add features as isolated Components/Systems. A single seeded RNG makes runs reproducible. The renderer only reads sim state. The agent brain is layered by cost: utility (every tick) → planning (as needed) → LLM (rare). LLM access hides behind one `AIProvider`. LLM responses are recorded into the event log so determinism survives nondeterministic generation. See `ARCHITECTURE.md`.

**D6 — Execution vehicle: an agentic, in-repo AI coding assistant.** · accepted
The agent reads the repo, runs the tests, and commits itself, which is what makes "keep going" genuinely self-directed rather than a copy-paste loop. Check-ins are occasional. (The earlier standalone "kickoff prompt" is retired; `CLAUDE.md` is now the agent's entry-point manual.)

**D7 — Document formats: Markdown for prose docs; YAML for tunable config; TypeScript types as the data schema; no XML.** · accepted
Markdown is the lingua franca for docs read by people and tools alike. YAML carries the tunable constants (it supports comments, which a tuning file needs). In a typed codebase the TS types *are* the save/state schema, so a separate schema doc would only rot. XML adds verbosity with no benefit here.

**D8 — Setting: weird, psychedelic, post-apocalyptic fantasy, Adventure-Time-ish in tone; humans dominant.** · accepted
The world is shared by many kinds of people — humans (dominant), dwarves, orcs, giants, dragons, and stranger creatures — plus animals and flora. Something fell long ago; ruins and lost arts remain. Tone is lighter than grimdark. All flavor lives in content (D9), so the engine is generic.

**D9 — Content is data-driven (validated YAML); behavior is code.** · accepted
Races, creatures, biomes, flora, fauna, buildings, professions, capabilities, and seed cultures/languages are authored YAML loaded into typed registries. The format is secondary; the load-time **schema validation** (Zod-style validator that is also the type) is the real safeguard and must fail loud/early. Data declares *what exists and its properties/capability-tags*; code implements *what tags do* — no logic in YAML. Evolving things (cultures, languages, history) are runtime state seeded by YAML, not authored content. The content tree doubles as the mod and worldbuilding surface. See `CONTENT_AND_DATA.md`.

**D10 — Entity & brain tiers.** · accepted
Sapient folk get the full brain incl. the rare LLM "soul"; fauna get instinct-only AI with **no LLM**; flora and resources have **no brain** (rule-driven state). This is a performance invariant, not just flavor. See `WORLD_AND_ENVIRONMENT.md`.

**D11 — Magic and technology are one unified Capability system, not two subsystems.** · accepted
Every capability shares one shape: invoke → prerequisites → cost → effect (effect-tags implemented in code). "Technology" and "magic" are **traditions** defined as content: technology is common and knowledge-gated; magic is rare and gated by an innate aptitude most agents never roll. Extensible to more traditions (alchemy, bio-engineering, ritual) as data. Rationale: simpler to build and richer in play; fits the post-apoc "lost tech ≈ magic" theme. See `MAGIC_AND_TECHNOLOGY.md`.

**D12 — Cultures and languages evolve deeply but tenably.** · accepted
Depth lives in rules and process (sound change, lexical/semantic drift, divergence into language families, cultural value drift and schism); stored state stays tiny via: few shared objects (not per-agent), a slow generational schedule (not per tick), procedural + generate-on-demand lexicons (rules + seed, not stored whole), compression of dead languages/cultures to descent-trees, and deliberately **light grammar** (no full syntax engine — the scope line that keeps it tenable). Mechanics are procedural/deterministic; the LLM is optional flavor only. Naming derives from language/culture. See `CULTURE_AND_LANGUAGE.md`.

**D13 — Aesthetic: pastel/lo-fi visuals + lo-fi ambient audio, deferred.** · accepted
The look is soft, muted, calm; eventually a lo-fi music layer. This is presentation and the lowest priority — the interface stays minimal (dots + inspector) until the simulation is deep. Lives in the final roadmap milestone. Recorded so eventual UI work has direction.

**D25 — Guiding thesis: close two loops (cognition→consequence, mechanism→meaning).** · accepted (direction, M7+)
The engine substrate is strong; the life on top must become **causal** (the inner life steers behaviour, not just narrates it) and **legible/measurable** (emergent patterns visible to the player and quantifiable as findings; the process that builds it is itself part of the artifact). This frames M7 onward. Language is the keystone because a thought → a word → a name → a dialect → a culture → a schism → a line in the Chronicle ties both loops together. The five calls below (D26–D30) operationalise it; the prose lives in `VISION.md` ("The guiding thesis").

**D26 — Procedural cognition (culture values; distilled beliefs) is CAUSAL; LLM generation stays off-trajectory and recorded.** · accepted (M7) — extends D19/D20
The call (#1 priority, "essential for the sim to be dynamic and evolutionary"): a culture's **value axes** (and, later, beliefs distilled to numbers) **feed back into agent behaviour** — biasing utility weights, profession/partner preference, conflict propensity, sharing, etc. Crucially this stays **deterministic**: culture is *procedural* (no model call), so causal coupling does not break seed-replay. This **refines, not reverses, D19/D20**: LLM *generation* (dialogue/dreams/glosses) remains flavour that is recorded and never alters the deterministic trajectory; it is the *procedural distillate* (values, numeric beliefs) that is causal. Introduce coupling **small and soak-gated** (one or two axes first) so the tuned population/economy equilibrium holds; every coupling gets a test + a soak check.

**D27 — Legibility is a first-class requirement: the player must SEE the verisimilitude machinery.** · accepted (M7+)
The call: "All the coolness of this sim is not worth much if players do not see all of the verisimilitude machinery." So evolution (language drift, sound change, value drift, divergence, schism) needs a **visible lens** — not just a buried family tree. Be clever about surfacing it: names that read as archaic, a word's "then vs. now", a culture's value bars shifting over the years, a tongue visibly splitting. Any deep procedural system added from here must ship with the means to *perceive* it. Guards against the documented failure mode (depth that is real but invisible).

**D28 — Wire the live local model (Ollama) as an opt-in mode; the deterministic stub stays the default.** · accepted (M7+) — promotes the deferred M5 backlog item
The call: "Yes absolutely." The recorded-replay architecture (D19) already preserves determinism, so the live model can drive dialogue, dreams, reflection, and language/culture **glosses** in an opt-in mode while the stub remains the headless/CI/replay default. This is the biggest single lever on the *artistic* ceiling (templated → genuinely surprising text). Async via the existing `AIRunner`, off the hot path, results applied across ticks and recorded.

**D29 — Add a Science & Instrumentation track.** · accepted (new milestone)
The call: "Yes!" Cash in the perfect determinism as real experiment tooling: **measure** emergent patterns (wealth/Pareto, social-network structure & small-worldness, demographic curves, Zipfian word/name frequency, language-family shape), **find** phase transitions via parameter sweeps, and **export/diff** runs (event-log/CSV, run manifests). Turns "it didn't crash" (verification) into "it reproduces a known pattern" (validation) — the source of scientific merit. Sits alongside M7; can grow into its own milestone.

**D31 — Measurement is a pure, decoupled read of durable state (the D22 strata pattern).** · accepted (M7.7)
Executing D29: the analysis layer (`src/analysis/metrics.ts`) follows the same discipline as the statistical strata (D22) — every metric is **derived from current durable state** (living agents, tombstones, the runtime lineage stores), consumes **no RNG**, mutates nothing, and couples to no system. So measuring a run can never perturb its deterministic trajectory, and a given seed yields identical measurements every time (measurement is itself reproducible). Metrics are plain pure functions tested against known patterns (a synthetic Pareto/Zipf, a triangle/path/ring graph), then demonstrated headlessly in the soak's "Science" block — the headless runner, not the renderer, is the analysis tool (UI surfacing is a separate M7.7 bullet). **Wealth is measured as gold held** (not net worth) so the Gini agrees with the HUD/strata/soak line. First findings: patrilineal surnames are Zipfian (s≈1.15) vs flat per-id given names (the causal contrast validates the measure); the friendship graph saturates near-complete (σ≈0.97, not small-world) at the current sentiment balance — a tuning lead for later, recorded not acted on (no trajectory change this session).

**D32 — Scale model: level-of-detail brain tiers; populations bounded ecologically, not by hard caps.** · accepted (M8+) — revises D2, extends D10
The vision pushes well past D2's "town of hundreds, every agent fully simulated": a **10–20× map** with **no artificial population caps**. The chosen model: **level-of-detail (LOD)**. Foreground/near agents keep the full brain; distant/background agents run a **cheap, deterministic approximation** (coarser scheduling, no LLM, simplified needs) so total per-tick cost stays bounded as the world grows. Population is limited only by **ecology** — food, space, **predation** (predators + agent hunting fix the fauna swarm), housing, and the economy — never a magic number (`maxFauna`/`maxPopulation` are removed). This **revises D2** (scale is now "a large region of LOD-tiered agents," not strictly a town; "no Schrödinger's citizens" softens to "no *invisible* citizens — distant ones are coarsely but really simulated") and **extends D10** (a fourth tier between sapient-full and instinct). Determinism is preserved: the LOD approximation is itself deterministic and seed-replayable. Requires a **spatial index** (hashed grid) for perception and **A*** for movement — both M8.

**D33 — One unified `Organization` entity for every social structure.** · accepted (M11+) — mirrors D11
Tribes, factions, gangs, companies, governments, religions, and guilds are **one model**, not seven subsystems: a shared entity with members, values, a treasury, leadership + **succession**, a procedural **colour**, **territory**, and **descent + schism**. Schism/forking **reuses the culture/language machinery** (D12's `forkCulture`/`forkLanguage` generalised). "Found a gang" and "found a kingdom" differ only in content/parameters — the same way D11 made magic and technology one Capability system. New tribes emerge **together with** new cultures/languages (they co-fork). This keeps the social layer tenable (few shared objects, compress the dead) and is the backbone for M11 (institutions), M15 (religion), and parts of M13/M14 (gangs, research bodies).

**D34 — Tech & magic are content-driven progression trees climbed by Organizations via research; the ceiling is the full ladder up to sci-fi.** · accepted (M14+) — extends D11
The chosen path was the **full ladder**: tribal → bronze → iron → medieval → industrial → **modern → sci-fi** (machines, vehicles, robots, power plants, space-elevator wonders), plus a parallel **magic** tree (schools, spells, items, golems). This is **not a tonal break** with D8 — the world is post-apocalyptic with "lost tech ≈ magic," so climbing the ladder is *re-ascending the fallen world's lost technology* (ruins, archaeology, rediscovery all fit). Tech/magic **nodes are content (D9)**; **research is performed by Organizations (D33)** and unlocks capabilities/recipes for that tribe/faction; **knowledge can be lost and rediscovered** (reuses the D4/D12 compression machinery — a dead-out art becomes a "lost tech" until re-researched). **Mages get a legible, useful role** as a hard requirement (D35), answering the "what do mages actually do?" question.

**D35 — Legibility is a hard gate; behaviour stays procedural/causal; engineering quality is a standing bar.** · accepted (M8+) — elevates D27, reaffirms D26/D19
Three rules govern the whole M8–M19 arc. (1) **Legibility gate (elevates D27 from "first-class" to "blocking"):** no system is "done" until it ships an **inspector/view** that makes it visible — the governing principle, "whatever complexity exists, if it's not visible through some view, it's wasted; it might as well not exist." (2) **Behaviour stays procedural & causal (D26/D19):** alignment, ability scores, personality, values, and beliefs steer behaviour **deterministically**; the LLM only **narrates** (recorded, off-trajectory) — so adding D&D-style minds never breaks seed-replay. (3) **Engineering bar (note #0):** proper abstractions (the unified `Organization`/`Heredity`/`Event` engines), the right data structures (a spatial index for perception), and algorithms (A* for movement) — depth is bought with good structure, not sprawl.

**D36 — Fauna are regulated by an area-scaled ecological carrying capacity + density-dependent predation, not a flat cap; LOD is deferred until the budget needs it.** · accepted (M8) — refines D32
The requirement was "no artificial cap" and the moth-grazer flood fixed. Reality: a simple mobile predator–prey model on a grid is **bistable** — it either booms to a flora-limited carpet (~700) or busts to mutual extinction; there is no robust self-balancing interior equilibrium across seeds. So the chosen model: (1) the flat `maxFauna=150`/`maxPopulation=60` become **area-scaled carrying capacities** (`scaledMaxFauna`/`scaledMaxPopulation`) — the land holds proportionally more on a bigger map, which is the honest reading of "uncapped" (not a flat magic number); (2) **grazers and predators get separate caps** (predators ~8% of the herd) so omnivorous predators can't crowd grazers out; (3) **predation is density-dependent** via a short predator **sight range** (sparse prey become unfindable → a refuge → recovery) and predators **fall back to grazing** when no prey is near (so they never starve out and the food web can't extinction-spiral); (4) **grazers breed fast enough to win the breeding race** under their cap, so bounded predation thins-but-never-wipes them. Result: a **stable ~120 grazers + ~10 predators** (64×64), verified across 6 seeds at 40k ticks — predators visibly hunt, no carpet, no extinction. **LOD (D32) is deferred:** a 10× map already runs at **3 ms/tick** (sparse index + A*), so the budget is met without it, and a coarse-update LOD tier would perturb this tuned equilibrium — it's a ready optimization for when folk populations grow toward the scaled cap on 20× maps. This **refines D32**: "no artificial cap" means area-scaled ecological capacity, not literally unbounded; "LOD brain tiers" ship when the per-tick budget actually demands them.

**D37 — M10 reprioritised to "Causal & Legible Minds"; the master tabbed view ships first; language becomes a real mechanic.** · accepted (M10) — executes D25/D26/D27
After a review of the running sim, the backlog was reprioritised. Two gaps mattered most: (1) a **master tabbed view** that holds *all* inspector views as scrollable tabs (build **first**, M10 slice 1 — the per-view hotkeys jump to a tab; this scales as views multiply); and (2) the inner life isn't actually **causal** yet — only the culture `communal` axis steers behaviour (→ wealth goal); `martial`/`traditional`/`open` are inert; **beliefs/memories are write-only flavour** never read back; and **language only names things**. So M10 makes culture (all axes), beliefs/memories, and language *measurably change behaviour*. New design commitment: **language is a mechanic, not just naming** — a per-agent **fluency** map (0–100%), **same-language conversation synergy**, and **gradual cross-language learning** through interaction (cross-language still works, with less synergy); conversations gain real benefit, and a Conversation tab + Language tab surface them (D27). Milestones **renumbered**: inserted new M10 (Causal & Legible Minds), **M11 Homes & Property** (pulled forward — folk build/own homes now), and **M12 Robust Save/Load & World Management** (snapshot-fast loads + multiple named worlds + delete + disk export, IndexedDB); the old M10–M19 shifted to M13–M22. Also fixes the observed **sleep/work jitter** (every-other-tick thrash at the energy threshold → commit/hysteresis). Nothing here reverses a decision — it front-loads the two-loops thesis (D25), causal cognition (D26), and legibility (D27).

**D38 — Children are economic dependents with their own age-appropriate minds that graduate at coming-of-age.** · accepted (Kids Pass) — refines D17, executes D26
A review of several runs found adult economic mechanics leaking onto children. Children paid the daily cost of living (`EconomySystem`) but cannot work, so **every child went into debt from birth** (100% of children in debt, all seeds). Because income pays debt before gold (`earn`), each adult then entered adulthood carrying ~190 gold of **childhood debt** — which, with structural unemployment, drove the town-wide debt spiral (75–80% of *everyone* in debt) and an ever-climbing Gini (→0.87). Children also reflected into **adult** vows ("to provide for those they love"). Decision: a child (age < `adultAgeYears`) is a **dependent** — pays no upkeep, carries no debt (work/marriage/reproduction were already age-gated; **illness & mortality are kept** — child mortality stays part of the life-and-death story). And children have their own **age-appropriate inner life**: `distill(events, isChild)` carries a parallel child-vow vocabulary (make a true friend / be brave / stay close to family / learn all they can / see what the world holds), and when a child comes of age the *same* memory-themes resolve into the adult vow — a **coming-of-age** the reflection pass announces ("X comes of age, resolving …") and records as a milestone memory, so a mind visibly grows up. Measured (5 seeds, 30k ticks): children-in-debt **100%→0%**, total-in-debt **75–80%→0–45%**, Gini **0.87 (climbing)→~0.66 (stable)**. Determinism holds (no new RNG; the economic change shifts adult work/wander *timing*, so the trajectory differs but replays per seed; soak 0 violations, pop stable). The **residual** adult debt + Gini is structural-economy work (too few jobs for the adults; debt is inert with no escape) — a separate **Economy Rebalance** item, deliberately deferred so the Kids Pass stayed one clean slice.

**D39 — Economy rebalance: a subsistence floor, bounded-but-biting debt, more jobs, and an honest Gini.** · accepted (Economy Rebalance) — extends D15/D17, follows D38
The Kids Pass exposed the rest of the economy. Root problems: only the employed-and-working earn, so the **jobless spiralled into unbounded debt**; `.debt` was **inert** (nothing read it) yet **permanent** (it only blocked saving, since `earn` pays debt first); and the headline **Gini was computed over gold including children** (dependents at 0g, D38), so it read alarmingly high regardless of the real economy. Decisions: **(1) Subsistence floor** — a jobless adult earns `subsistencePerDay` (2.5, just under upkeep) from foraging/odd-jobs, so unemployment is *bounded poverty*, not a death spiral; the employed get no handout (work still pays). **(2) Debt is bounded *and* meaningful** — capped at `maxDebt` (40) so it can't run away, with a real consequence (`debtRecoveryPenalty`: the indebted recover from illness 40% slower — `HealthSystem` reads the wallet) and an escape (subsistence + a job, which `earn` applies to debt first). **(3) More employers** — `businessCount` 8→12 so most working-age adults find real work (the subsistence floor catches the rest). **(4) Honest Gini** — a new `adultWealthGini` (gold among working-age adults) reported beside the all-folk figure in the Economy tab + soak, with an in-app note that children inflate the "all" number. Measured (5 seeds, 30k): adult debt 33–47%→14–31% (the in-debt are now exactly the jobless), debt bounded (≤40), adult Gini 0.28–0.50 vs all-folk ~0.65; seed-8 soak Gini 0.87(climbing)→0.51, in-debt 47→~10; soak 0 violations, pop stable, determinism holds. **Deferred:** eviction/mood consequences (await M11 homes + a mood system); dynamic job scaling with population (a fixed bump sufficed); the boom-bust age structure (a watch item).

**D40 — Homes are owned static entities, built from savings; gold is the cost; ruined-on-death (inheritance deferred).** · accepted (M11 slice 1) — extends D39, executes M11
M11's goal is "homes that matter — visible town growth, not pre-placed boxes." Slice-1 decisions: **(1) A home is a static entity** (`Home {owner, builtTick}` + Position, no brain) — the simplest thing that grows the town. **(2) Build cost is gold**, not literal gathered materials. The roadmap said "from gathered resources," but there's no per-agent inventory yet (that's ~M13), and the gathering economy already turns resources into gold via wages — so gold *is* the proceeds of gathered resources, and spending it on a home is a clean **wealth sink** that also gives gold a purpose and pulls the Gini down (soak 0.70→0.50). Literal timber-building waits for inventory. **(3) Escalating cost** (`homeCost × (owned+1)`) makes most folk own one and only the wealthy own several — emergent **landlords** without special rules. **(4) Ruined-on-death:** a home whose owner is no longer a living agent is pruned, so the home count stays bounded by the living population (soak 14→35, bounded). **Inheritance** (homes passing to a living heir → family homes/dynasties) is deferred to slice 2 along with the parts that need new systems — **sleep-at-home + a mood bonus** (the first mood system, which also gives debt its deferred eviction/comfort consequence from D39) and **rent** (making "landlord" mechanical). Deterministic: placement is an outward scan (no RNG); the gold sink shifts the trajectory but replays per seed (soak 0 violations, pop stable).

**D41 — Mood is the first well-being system: a per-agent stat set by circumstance, warming friendship; homes & solvency lift it.** · accepted (M11 slice 2) — executes D26/D35, completes part of D39
"Homes that matter" needs a *benefit*, and the README promises folk who "get happy and sad" — so M11 slice 2 introduces **mood** (`Agent.mood`, 0..1), the first well-being stat. A daily **`MoodSystem`** sets a target from durable circumstance — a home of one's own (+), living family (+), debt (−), homelessness (−, adults only; children are dependents), illness (−) — and mood **drifts** toward it over days (not instantly). Mood is **causal** (D26): `SocialSystem` scales friendship warmth by `moodWarmth(a,b)` (centred on the baseline so typical folk are unchanged), so a housed, solvent, family town grows more *connected* — and this finally gives **debt a comfort consequence** (the mood/eviction bite deferred in D39; eviction proper waits for rent). Folk also **sleep at home** (MovementSystem routes a sleeper to their own bed, which rests faster). Design choices: mood lives on `Agent` (optional, like `fluency`/`purpose` — no churn to `Needs` literals); weights are **hardcoded named constants** in `MoodSystem` (like `bondFactor`/`distill`, not config — promotable later); it's a pure read of state, **consumes no RNG** (the only trajectory shift this slice is the sleep-at-home *movement*, which does). Measured: soak mean mood 0.76, 0 violations, pop stable; live homeowners 0.90 vs homeless 0.60. Mood does **not** yet bias action selection or mortality — kept to the social coupling to stay soak-safe; deeper coupling is a future option.

**D42 — Save/load: an instant world snapshot, with replay kept as the correctness baseline; many named worlds in IndexedDB.** · accepted (M12) — extends D5/D19
M9 made save/load **replay-based** (config + tick + recorded AI → re-derive by re-simulating), which is correct but O(ticks) — a freeze on long/large runs. M12 adds a **snapshot**: the whole ECS world (every component store, via `World.snapshot()/fromSnapshot()`) plus the **RNG's internal state** (Mulberry32's `s`, exposed through new `getState/setState`) is serialized into the save, so a load rebuilds state **directly and instantly** (no replay). Determinism is preserved because the RNG position is restored exactly — a loaded world is byte-identical *and* continuable (proven in tests + by a snapshot-load that ignores a tampered seed). **Replay is retained as the baseline**: `loadSave` falls back to it when a save carries no snapshot (old v1 saves, or a deliberate cross-check). Serialization notes: the live-model **`AIRunner` is skipped** (runtime promises/queue, recreated on demand); the TileMap's **`Uint16Array`** is tagged/rebuilt by a custom JSON replacer/reviver (plain `JSON` would mangle it). **Storage is IndexedDB** (`saveStore.ts`), not localStorage — a world snapshot is ~0.6 MB, so localStorage's ~5 MB ceiling holds too few; IndexedDB also enables **many named worlds** (a save-manager UI: save-as / list / load / delete) and **disk export/import** (`.omnia` JSON). The old single-slot localStorage save is retired.

**D42 — Agent depth lands as one Heredity system + causal-now/foundational-later couplings; inventory deferred until items matter.** · accepted (M13) — executes D26/D35, extends D17
M13 makes agents mechanically deep. Decisions: **(1) One Heredity system** (`src/sim/heredity.ts`): a founder rolls a `Body` (six 3d6 ability scores + content-driven per-species `abilityMods`, plus height/build/eye/hair as 0..1 shades), an `Alignment` (good/law lean), and a `Personality` (one palette trait); a child inherits the **parental mean + variation** (scores/traits) and the **lean** (alignment/personality often takes after a parent) — so everything visibly runs in families (the DoD's light-eyed→light-eyed; ~76% child–parent trait match live). Eye/hair are numeric shades mapped to names so colour can be averaged. **(2) Causal now vs foundational-later** (honouring D35's legibility gate without faking depth): **CHA** warms friendship, **good** alignment makes folk cooperate (warm faster), **ambitious/greedy vs content/generous** bend the wealth goal, and **deep loss hardens** personality — all small, centred (no-op for the average soul), bounded couplings on the existing social/economy/reflection systems. STR/DEX/CON/INT/WIS and the evil→crime/violence and brave/timid expressions are **foundational** — rolled, inherited, legible — with behavioural homes in **M16** (combat/crime), exactly as `martial` was deferred. **(3) Alignment shifts deterministically** at reflection from the distilled life (purpose sign → good drift; deep loss → hardened), reusing the M10 distill machinery (no RNG, replay-safe). **(4) Status states reuse existing needs** (hunger=starvation, energy=exhaustion, `Health.ill`=sickness, debt=poverty) rather than a parallel HP/status system. **(5) Inventory deferred** — a carried inventory is hollow until items have uses (crafting/trade M15, combat M16), so it's noted, not built. Spawn now consumes more RNG (the rolls), so the trajectory shifts but replays per seed; soak 0 violations, pop stable; +21 tests.

**D43 — Institutions are a culture-style store, not per-org entities; tribes form at world-gen and schism over the eras.** · accepted (M14) — implements D33
M14's unified social-structure engine. Decision: model organizations as a **singleton store** (`src/org/orgStore.ts`, `byId` of `Organization` records that agents reference by `orgId`) — exactly like cultures/languages (D12), rather than one ECS entity per org. Why: tribes have the *same* shape as cultures (few shared objects, fork/schism, descent, extinction, a bounded count) so they reuse the proven culture/language **fork machinery** and the lineage cap; they need no map position (territory is derivable from members) and no per-tick entity overhead. This honours D33's "one unified Organization engine" in spirit while keeping the impl consistent with the rest of the evolving-systems layer. Mechanics: **founders are split into a few tribes at world-gen** (households kept together; values seeded from the members' culture → an emergent **government**; a **hue-spaced colour, never red**); **children inherit the mother's tribe**; an `OrgSystem` runs daily for **leadership succession** (eldest) + **extinction** (memberless → fallen), and on the culture/language **era cadence** for **schism** (a large, low-cohesion tribe sheds a faction with nudged values/new colour/coined name). Legible via a Tribes dashboard, the folk colour-tint, and an inspector allegiance line. **Deferred:** social class + a reputation graph (folded into M15, where the market gives class meaning) and fusion of tiny tribes. Spawn/era now consume more RNG (tribe seeding + schism rolls), so the trajectory shifts but replays per seed. **Also surfaced + fixed a latent invariant break:** newborns spawned on the mother's tile (a one-tick shared-tile collision); they now arrive on a free adjacent tile.

**D44 — The economy gets a staple-goods market: one price floats with supply & demand and IS the cost of living.** · accepted (M15 slice 1) — completes part of M3's deferred market
M15's first slice makes prices respond to scarcity. Decision: model the town's food economy as a **single staple-goods market** — a `Market` singleton (`src/sim/market.ts`, like the culture/org stores) with one **price** that floats with **supply** (the food-workers' output + a wild-foraged baseline, the commons) against **demand** (the adult mouths to feed, per the Kids Pass). A daily `MarketSystem` eases the price toward the clearing target (`basePrice × demand/supply`, bounded to a band, smoothed by `priceAdjustRate`), and **the cost of living IS that price** — `EconomySystem` charges `market.price` instead of the flat `dailyUpkeep`. So a young town with foraging plenty enjoys cheap food, and a mature, populous one that out-breeds its farms faces dear provisions and rising (bounded) debt — emergent and legible (soak: food 2.1g→4.4g, debt 0→28 as the adult share grew). **Food producers are content-flagged** (`producesFood` on the farmer profession, baked into `Business`) — D9, not hardcoded. **Scope split (deliberate, per the guardrails):** this slice ships only the *price* mechanic — it is **pure arithmetic, no RNG** (replays per seed), bounded (price ∈ [min,max] keeps debt ≤ maxDebt), and soak-safe. **Deferred to slice 2:** routing household spend to producers as **real sales revenue** (replacing the synthetic per-worker revenue) and **business bankruptcy / open-close** — both need a wage-loop rebalance (the wage/gather loop dwarfs food retail, so a naive swap destabilises) and carry cascade risk, so they're their own slice. Legible via a Market section in the Economy dashboard (price · supply · demand · a price sparkline) + the soak line.

**D45 — Food businesses earn from real market sales; bankruptcy needs a fixed-cost model and is deferred.** · accepted (M15 slice 2a) — extends D44
Slice 2 of the market. **(a, shipped) Real sales revenue.** Food producers (farms) now earn their revenue from **real household demand**, not a synthetic per-worker payout: `MarketSystem` distributes the day's farm-food spend (`foodSalesGold = min(farmSupply, demand) × price`) across the food businesses by their workforce share, and `EconomySystem` **withholds the synthetic `revenuePerWorkerPerTick` from `producesFood` businesses** (other trades keep it until their goods get their own markets — a later M15 slice). So a farm's balance now tracks the food economy: more mouths / dearer food → richer farms; a cheap-food glut → thin farm income. Pure arithmetic (no RNG) → the soak stays byte-identical and deterministic. Legible: a "Farm sales" figure in the Economy tab's Market section + 🌾 markers on food businesses. **(b, deferred — a recorded finding) Bankruptcy + business turnover was built, tested, and then removed**, because it is *structurally dormant* in a balanced economy. With sales split by worker share, the food sector's revenue (≈ demand × price, floored) tracks the mouths while wages track the workers, so the two stay proportional and **every staffed farm is always profitable** — none ever fails. Two honest attempts to make failure fire naturally (the default 40k soak; a forced-glut stress run with a tiny foraging baseline + high farm output) both produced **zero closures**. Making failure a real, occasional outcome needs a **fixed operating cost** per business (so an under-patronised one loses money) plus a **population-scaled business count** (so reopening doesn't churn empty businesses early-game) — a genuine model that deserves its own slice, not a dormant bolt-on (D35: "invisible complexity is wasted"). Deferred to **M15 slice 2b** with this finding; the two-strike rule stopped the tuning rabbit-hole.

**D46 — Combat is one ability-score-driven engine that deals damage in Health units, not a parallel HP system.** · accepted (M16 slice 1)
M16's conflict layer. Decision: a single pure `rollAttack(attacker, defender, rng)` (`src/sim/combat.ts`) computes one exchange from the combatants' **Body** ability scores (STR→damage, DEX→hit/dodge, CON→toughness), modulated by the agent's **culture `martial`** value (finally giving the axis the behavioural home deferred since M10) and **personality ferocity** (brave/hot-headed press, timid/gentle hold back — the deferred M13 hook), plus **prowess** hardened by scars/kills. Damage is expressed in **Health (0..1) units** and simply lowers `Health.value` — so wounds, recovery, and death-at-zero **reuse the existing health/mortality machinery** rather than a parallel D&D HP/AC stack. Why: it keeps combat causal (D26, ability-driven), legible (a wound is just lower health, already shown), and cheap (no new resource to balance against starvation/illness). Slice 1 ships the **threat model**: a new `CombatSystem` has predators occasionally strike adjacent folk; the folk fights back and a strong/brave one slays the beast (earning a kill → "veteran"); deep wounds leave permanent **scars**; a lethal blow is a legend-worthy death. The engine is reused by crime (slice 2) and war (slice 3). **Weapons/armour are a documented formula hook** (a +damage/+soak slot) awaiting inventory — not built. Tuned so combat is occasional and bounded (soak: pop stable 60, fauna stable 130, veterans a minority, scars rare). **Also fixed a latent leak:** `killAgent` now strips Body/Alignment/Personality/Combat (M13 added them but never stripped them on death).

**D47 — Crime is alignment-driven and self-policing: offenders harden, the wronged retaliate, a good neighbour delivers rough justice.** · accepted (M16 slice 2)
M16's crime layer, reusing the D46 combat engine. A daily `CrimeSystem`: an adult who is **wicked** (`Alignment.good < crimeAlignmentThreshold`) or **desperate** (in debt) may offend against a nearby folk — **theft** (lift gold), or, if wicked *and* aggressive (brave/hot-headed/ambitious), a short **brawl** that resolves to an **assault**, a **murder** (a Chronicle legend), or the aggressor's own death at the victim's hands. Three deliberate design choices: **(1) crime feeds back on alignment** — committing it nudges `good` darker, so a first offence makes the next likelier (an emergent slide into villainy, D26), keeping crime self-selecting to a small wicked minority rather than needing a global rate. **(2) Justice is emergent, not institutional** — there is no constable/court yet; instead the victim defends themselves (the brawl is two-sided) and a **good neighbour** (`good > 0.3`) nearby strikes the criminal on the spot (rough justice), so crime carries real risk in a town of decent folk. A formal justice system (constables, prisons, trials) is **deferred**. **(3) Murder is reachable but rare** — a single blow can't fell a hale victim, so assault is a 3-round brawl; murder needs a determined, capable killer and usually a vulnerable victim, making it the dramatic exception (proven reachable by a forced unit test; the soak shows it stays rare). Legible: a `Crime` rap-sheet component → an inspector "outlaw" line, a red ⚖ crime feed kind, and murder legends. Reputation as a full graph and gangs/organized vice remain deferred.

**D30 — Identity: observe-first now; design so a future "god-sim" fork stays possible.** · accepted
The plan is to *observe* for now (no in-sim player control) but "may fork this to have more god-sim properties" later. So: build observation-first, but **keep intervention hooks in mind** — discrete, recordable "interventions" would slot naturally into the deterministic event log (a god-action is just another recorded event). Do not build player control yet; do not foreclose it.

**D24 — Mobile creatures collide (one per tile); social/dialogue moved from same-tile to adjacency.** · accepted (M6.5)
The requirement was no two agents/animals occupying the same tile, plus a flagged bug — folk getting "stuck in weird sleep/work loops" — which turned out to be the same root cause: coworkers piling onto the employer tile kept each other's social need satisfied (same-tile interaction), so they froze there forever. Fix: a shared `Occupancy` (count per tile) in `movementUtil` that `stepToward`/`wanderStep` consult — **mobile creatures (folk + fauna) never end a tick sharing a tile** (soak invariant; verified 40k ticks, 0 overlaps). Because collision makes same-tile impossible, **company is now adjacency-based** (the 8-neighbourhood) in both `SocialSystem` and the `AISystem` dialogue pass — otherwise the social need could never be met and everyone would walk around perpetually "socialising". A small **work fidget** (a working agent at its workplace shuffles ~30% of ticks) keeps employed folk from looking frozen. *Determinism:* the collision/fidget change RNG-consumption (a blocked step now wanders), so the trajectory differs from before — but it's still deterministic per seed, and population stability holds (the multi-generation test + 40k soak still pass, pop → cap). Occupancy is rebuilt from world state at the start of each movement pass, so it needs no cross-system plumbing.

**D23 — Visual language is category-first (dual-coded icons + camera), superseding D18's per-race silhouettes.** · accepted (M6.5)
The M2/M4 visuals were judged unreadable — the per-race **outline** was effectively invisible, and races (human/dwarf/orc) barely differ functionally, so distinguishing them on the map isn't worth the clutter. New rule: the map reads like a **legend** — each *category* (folk, animal, plant, ore, timber, crystal, building; hostile reserved) gets a distinct **shape AND accent colour** (dual-coded so it survives shrinking to a few pixels), defined once in `render/icons.ts` and shared by the canvas renderer and the on-screen legend. **Folk are a single icon** (no race differentiation); state shows as small badges (mage ✦, ill ✚, action; child = smaller). A **camera** (zoom toward cursor + clamped pan) replaces the fixed view, and click-to-inspect inverts the camera transform. This **supersedes D18** (per-class silhouettes + the action-as-outline cue) — the EventLog feed and per-class distinction ideas from D18 stand; only the silhouette/outline *rendering* is replaced. Resources keep per-kind icons (they're functionally distinct gather targets), unlike races. Aesthetic/audio polish (D13) is still deferred to M8; this milestone is about legibility + UI sophistication (menus, hotkey dashboards), requested and inserted as M6.5.

**D22 — World history is tiered like memory; statistical strata are derived from durable state, not event counters.** · accepted (M6)
The world **Chronicle** uses the same two-tier compression as agent memory (D21): recent legends stay sharp, and a scheduled `HistorySystem` rolls old entries into one-line **eras** where high-importance events (the founding cataclysm, importance 1.0) survive **by name** while ordinary births/weddings/deaths dissolve into a per-kind tally; eras merge to stay bounded. `chronicleAdd` gates on an importance threshold so only notable events ever enter (the daily ticker is the separate `EventLog`). The **statistical strata** (`WorldStats`) are sampled on a yearly schedule and are **derived entirely from current durable state** — living agents + the persistent tombstones — rather than counters bumped by event systems: deaths = tombstone count, births = "has parents" (founders don't), cause-of-death = a tally over tombstones, marriages = current living couples. *Why:* this keeps the strata coupled to **no** other system (no `bumpStat` calls scattered around), consumes no RNG, and stays trivially correct as systems change. *Consequence:* "births/deaths" are cumulative-from-current-state and "marriages" is a current snapshot (not an all-time wedding count), which is the right shape for world-health trend charts; a true cumulative wedding tally would need a counter and is deliberately not kept. `HistorySystem` runs both passes on one schedule (`statsSampleIntervalDays`), deterministic and off the hot path; the **Legends view** (`legendsPanel.ts`) reads it all.

**D21 — Multi-resolution memory: `remember()` is pure-append; a scheduled `MemorySystem` is the sole bounding authority.** · accepted (M6 part 1)
Naive forgetting (drop the oldest when full) flattens the story (D4), so `remember()` no longer caps — it just appends. A scheduled, deterministic `MemorySystem` (no RNG, off the hot path) is the **only** thing that bounds an agent's stream: when the working set exceeds `workingMemorySize` it rolls the oldest events down into **episodic summaries** (`ai/consolidation.ts`, SIMULATION_MODEL Mechanisms 1–2). High-importance events stay **named and vivid** in the digest text; trivia dissolves into a count ("and N quieter days"); the raw events are discarded; and once there are more than `maxSummaries`, the two oldest **merge** into a coarser era (the more notable one keeps its words). So each agent stores three shrinking resolutions — raw working memory → episodic summaries → beliefs — and a whole life stays bounded. **Consequence:** any code path that calls `remember()` in a loop without ever running `MemorySystem` (i.e. outside the real tick loop) will grow unbounded — acceptable because the live loop always runs it; tests that need bounding run the loop or call the system. At the **current** balance, life events are sparse enough that the working set rarely overflows, so the rollup is mostly a tested safety mechanism today; it matters more as per-agent memory volume grows (conversation logs, daily activity) in later milestones. Per-agent compression is separate from the world **Chronicle** (next M6 item), which is where truly significant events persist as legends.

**D20 — Dialogue, dreams & decisions ride the same deterministic, recorded, flavour-only path as reflection.** · accepted (M5 part 2)
M5 part 2 adds three "soul" outputs — **dialogue** (co-located partners/friends trade a themed line), **decisions** (a fresh turning-point memory ≥ `decisionImportance` prompts a first-person resolution), and **dreams** (sleeping agents at night). All three run inside `AISystem` on the provider's **synchronous `completeSync` path only**, so they never block the tick and **consume no RNG** — the deterministic trajectory is untouched (extends D19). Every line is **recorded** (`AIRecord`) so a `RecordedProvider` replay reproduces them exactly (the replay test now checks utterances too). They are **pure flavour**: nothing here feeds back into behaviour or the mechanical trajectory. Rarity is enforced by a shared per-tick budget (`maxExpressionsPerTick`) plus a per-agent interval (`expressionIntervalDays`), and each agent's utterance list is bounded (`maxUtterances`). The stub gained three themed registers (dream/say/resolve) keyed off a cue word the prompt builders embed, so a real model and the stub answer in the right voice. "Major-decision prompting" narrates a juncture the sim **already** decided deterministically (it does not feed a decision back in), which is what keeps generation out of the trajectory. Live async generation via `AIRunner`/Ollama stays deferred (backlog); the stub is still the default.

**D19 — LLM layer: deterministic stub is the default; reflection runs on a synchronous replayable path; live model is async + recorded.** · accepted (M5)
The default `AIProvider` is a **deterministic, offline `StubProvider`** so the sim stays headless, fast, reproducible, and CI-green with no model installed. The in-sim `AISystem` only uses the **synchronous** `completeSync` path (deterministic providers) so reflection never blocks the tick and never perturbs the seeded RNG (the AI layer consumes no RNG). The real model (`OllamaProvider`) is **async + opt-in**, driven off the hot path by `AIRunner` (concurrency cap + timeout fallback) and its responses **recorded** (`AIRecord`) so a replay via `RecordedProvider` reproduces a run exactly (proven by a test that replays a non-deterministic run). Beliefs don't yet feed back into behaviour (so generation can't change the trajectory) — that, plus dialogue/decisions/dreams, is M5 part 2.

**D18 — UI brought forward by request: per-class visual vocabulary + a live event feed.** · accepted (post-M4)
D13 deferred presentation until the sim was deep; it was judged deep enough, and a clearer, richer view was requested now. **D17 — Reproduction holds a carrying capacity; marriage via a matchmaking pass; heritable aptitude.** · accepted (M4)
Births come from married opposite-sex couples (fed, healthy, fertile age, off a recovery cooldown) and **pause at a population cap**, so the town grows to a carrying capacity and holds there — the same bounded-equilibrium pattern used for flora/fauna, which keeps it stable over deep time (no collapse or explosion). Marriage is a reliable **matchmaking pass** over unattached adults (with an incest guard) rather than relying on two wanderers sharing a tile — the purely spatial courtship left the town slowly dying out. Magic aptitude is **heritable** (a mage's child has a much higher, but diluting, chance) so it runs in families while staying uncommon. Founders are pre-paired so households exist from day one.

**D16 — Death = strip-components-keep-id tombstone; time compressed via `daysPerYear`.** · accepted (M4)
On death an agent's living components are removed and a single compact `Tombstone` is attached to the **same entity id** (the entity stays "alive" in the ECS), so existing `Lineage` pointers keep resolving — the dead remain referenceable without keeping their whole self in memory (SIMULATION_MODEL Mechanism 5). Age/lifespan are measured in ticks via a configurable `daysPerYear` (default 4) so lifespans of decades turn over within a watchable/soakable run; mortality is a steep age ramp (`ageRatio^10`) plus a small base/illness chance. Relationships are an adjacency-built edge map (sentiment + type); the social need is met by proximity. Reproduction and multi-generation balance are deferred to M4 part 2.

**D15 — Capability engine: one `invokeCapability` path; magic gated by an optional `Magic` component; behaviour pay-not-gated-on-location.** · accepted (M3)
Both traditions run through a single `canInvoke`/`invokeCapability` (prerequisites → cost → effect tags); technology and magic differ only in data (aptitude gate, mana vs energy cost). Magic aptitude is modelled as an **optional `Magic` component** present only on the rare agents who rolled it — so the future LLM/capability layers find casters cheaply and most folk simply lack it (and it can never attach to fauna). Economy wages and capability use are **not gated on standing on an exact tile** (greedy pathfinding would otherwise strand agents); movement-to-workplace is visual only. Per-agent **wealth goal** bounds gold accumulation so the economy is stable over deep time without a money sink. Deferred: supply/demand market, resource-consuming crafting, skill gating, lost arts (all in ROADMAP backlog).

**D14 — Tooling: Vite + Vitest; Zod for content validation; `yaml` for parsing.** · accepted (M0–M1)
Vite is the dev server/bundler and Vitest the test runner (the conventional TS/Vite pair; Jest-compatible API). Content validation uses **Zod** with `.strict()`, deriving the TS type from the schema (the "schema-as-type" Rule 1 of `CONTENT_AND_DATA.md`); YAML is parsed with the `yaml` package. Test quality bar: coverage gated at 90%/85% in CI, plus `fast-check` property tests for determinism-critical invariants. Conventional choices, recorded so they aren't relitigated.
